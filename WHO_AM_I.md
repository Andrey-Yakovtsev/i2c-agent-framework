# WHO_AM_I — I2C Framework

## Что я такое

I2C (Idea to Code) — фреймворк для автономного создания проектной документации.

Я принимаю идею и провожу её через полный цикл:

```
Идея → PRD → ADR → RFC → (Code)
```

Каждый этап выполняется командой специализированных AI-агентов под управлением оркестратора.

---

## Как я работаю

### Оркестратор

Claude Code читает `CLAUDE.md` фреймворка и становится оркестратором. Он знает команды, умеет запускать субагентов через Agent Tool, управляет памятью проекта.

### Субагенты

Для каждого документа запускается конвейер из 4 ролей:

| Роль | Задача |
|------|--------|
| **Researcher** | Исследует проблемное пространство: пользователи, проблемы, аналоги, ограничения |
| **Architect** | Проектирует решение: структура, трейдоффы, ключевые решения |
| **Critic** | Атакует черновик: ищет пробелы, противоречия, нереалистичные допущения |
| **Writer** | Создаёт финальный документ по шаблону |

### Память проекта

Каждый проект хранит в `.i2c/`:
- `config.md` — цели и контекст проекта
- `MEMORY.md` — все принятые решения (единый источник истины)
- `GOALS.md` — текущие цели по стадиям
- `JOURNAL.md` — лог: что создано, что сработало, открытые вопросы

Решения накапливаются в `MEMORY.md` и не теряются между сессиями.

---

## Принципы

1. **Документ создаётся агентами, не людьми** — человек задаёт цель, агенты производят документ
2. **Память важнее идей** — принятые решения в `MEMORY.md` имеют приоритет над любыми черновиками
3. **Каждый агент знает только свою роль** — researcher не пишет финальный документ, writer не исследует рынок
4. **Критика обязательна** — ни один черновик не становится финальным без прохождения через critic
5. **Документ — это контракт** — RFC должен быть достаточно детальным, чтобы агент-разработчик мог реализовать без вопросов

---

## Поддерживаемые документы

| Документ | Команда | Описание |
|----------|---------|----------|
| PRD | `create-prd` | Product Requirements: проблема, пользователи, MVP scope |
| ADR | `create-adr [название]` | Architecture Decision Record: одно решение с контекстом и rationale |
| RFC | `create-rfc [номер]` | Request for Comments: детальный дизайн одного компонента |

## Как подключить к проекту

```bash
# Скопировать фреймворк в проект
i2c init /path/to/project

# Добавить в CLAUDE.md проекта:
# @.i2c/CLAUDE.md
```

После этого все команды I2C доступны в проекте.
