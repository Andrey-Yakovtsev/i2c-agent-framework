Обновить реализацию компонента после изменения RFC.

Анализирует дельту между текущим RFC и оригинальным IMPL-планом,
запускает coding-агентов только для изменённых AC и интерфейсов,
прогоняет полный тест-сюит (включая существующие тесты — проверка регрессий),
верифицирует покрытие новых AC.
Флаг `--auto` отключает запросы подтверждений.

**Аргумент:** номер RFC (обязательно).
**Пример:** `/i2c-patch-rfc 3` или `/i2c-patch-rfc 3 --auto`

Если `$ARGUMENTS` пуст — спроси пользователя: "Введи номер RFC для патча (и опционально --auto), например: '3' или '3 --auto':"
Дождись ответа и используй его как аргумент.

Выполни команду I2C `patch-rfc $ARGUMENTS` из инструкций оркестратора.
