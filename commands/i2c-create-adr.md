Создать Architecture Decision Record для конкретного архитектурного решения.

Запускает: Supervisor → Architect → Critic → Writer → Supervisor.
Результат сохраняется в `docs/ADR-NNN-slug.md`.
Использовать: когда решение трудно отменить, есть значимые трейдоффы, решение влияет на несколько RFC.

**Аргумент:** название решения (обязательно).
**Пример:** `/i2c-create-adr Выбор базы данных` или `/i2c-create-adr Auth strategy`

Если `$ARGUMENTS` пуст — спроси пользователя: "Введи название решения для ADR (например: 'Выбор базы данных', 'Архитектурный стиль API'):"
Дождись ответа и используй его как аргумент.

Прочитай файл {{FRAMEWORK_DIR}}/orchestrator.md (используй Read tool) и выполни команду `create-adr $ARGUMENTS` по инструкциям из этого файла.
