Реализовать компонент по RFC через конвейер coding-агентов.

Создаёт Implementation Plan (Architect + Critic + Writer), поднимает Docker-окружение,
запускает coding-агентов и test-writer агентов параллельно, прогоняет тесты, верифицирует AC.
Флаг `--auto` отключает запросы подтверждений у пользователя (агенты работают автономно).

**Аргумент:** номер RFC и опционально `--auto` (обязательно).
**Пример:** `/i2c-code-rfc 3` или `/i2c-code-rfc 3 --auto`

Если `$ARGUMENTS` пуст — спроси пользователя: "Введи номер RFC для реализации (и опционально --auto для автономного режима), например: '3' или '3 --auto':"
Дождись ответа и используй его как аргумент.

Прочитай файл {{FRAMEWORK_DIR}}/orchestrator.md (используй Read tool) и выполни команду `code-rfc $ARGUMENTS` по инструкциям из этого файла.
