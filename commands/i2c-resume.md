Продолжить прерванный пайплайн с того шага, где он остановился.

Читает `.i2c/pipeline_state.json`, показывает завершённые и оставшиеся шаги.
Предлагает варианты: resume (продолжить), restart (начать заново), abandon (отменить).

**Аргументы:** не требуются.
**Пример:** `/i2c-resume`

Прочитай файл {{FRAMEWORK_DIR}}/orchestrator-installed.md (используй Read tool) и выполни команду `resume` по инструкциям из этого файла.
