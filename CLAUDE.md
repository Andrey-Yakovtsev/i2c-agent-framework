# I2C Agent Framework

Documentation orchestration framework. Install into a project: `./install.sh /path/to/project`

## Правила разработки оркестратора

### Лимиты размера

| Файл | Максимум |
|------|----------|
| `orchestrator-source.md` | 2,000 слов |
| Каждый `protocols/*.md` | 500 слов |
| Новая команда в оркестраторе | 100 слов (остальное — в protocol) |

### Принципы

- **Не дублируй:** паттерн используется 2+ раз → извлеки в `protocols/`
- **Только дельта:** команда описывает ТОЛЬКО отличия от базового протокола
- **Стандартные входы не перечисляй:** MEMORY.md передаётся всегда (указано в конвенциях)
- **pipeline_state не описывай пошагово:** ссылайся на конвенцию
- **Протоколы используют относительные пути** к агентам (`agents/X.md`), оркестратор задаёт базовый путь

### Перед коммитом

Проверь: `wc -w orchestrator-source.md` ≤ 2000, `wc -w protocols/*.md` каждый ≤ 500.
